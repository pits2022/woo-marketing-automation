# woo-marketing-automation Wordpress plugin development

## Current state

Currently the PHP scripts in this repo has 2 main functions:

1. Subscribe visitors to Sendy
2. Sending customer reactivation e-mails

Customer reactivation e-mails are scheduled via external cron.

Configuration and sensitive information is stored in `.env` file that creates a potential security issue.

## Wordpress plugin development

The development of a Wordpress plugin based on the current scripts and templates has significant benefits:

* Easy installation
* Sensitive data is stored in Wordpress database tables (`wp_options`)
* The usage of `wp-cron`
* Seamless WooCommerce integration

## Technical Architecture & Guidelines

* **Admin Menu:** The plugin's settings page should be embedded within the **WooCommerce** admin menu (e.g., `WooCommerce -> Marketing Automation`).
* **Frontend Submissions:** The subscription form must use WordPress's built-in AJAX architecture (`admin-ajax.php` or WP REST API). Forms generated by shortcodes must use dynamically generated unique IDs/classes to prevent collisions when multiple forms are present on the same page.
* **Cron & Scheduling:** The plugin uses `wp-cron` for scheduling reactivation emails. 
  * *Note for Server Admins:* To ensure reliable delivery, a server-level cron job should be configured to trigger `wp-cron.php` regularly.
* **State Tracking:** Email dispatch states must be tracked using WooCommerce order meta (e.g., `_review_email_sent`), ensuring emails are not sent multiple times per order.
* **Sendy API Calls (Async):** API calls to Sendy (such as adding customers to lists upon order completion) must be executed asynchronously (e.g., using non-blocking `wp_remote_post` or a single scheduled WP-Cron event) to prevent slowing down the WooCommerce checkout process. Data should be read from WooCommerce order meta whenever possible to minimize external API requests.
* **Legacy Logic Removal:** The legacy `allowedRcpt()` (TLD filter) and the hardcoded `nyeremenyjatek` logic are completely removed. The dynamic shortcode parameters (`list` and `redirect`) natively handle different subscription scenarios.

## Functions and features of the Wordpress plugin

1. Custom Sendy subscription via Wordpress short code
2. Sending custom customer reactivation e-mails and welcome e-mails
3. Adding new customers to customer lists (customers, VIP customers, returning customers)
4. English and Hungarian language support via `.po` files
5. Global HTML Email Template Management

### Global Email Template Manager

The plugin will provide a built-in HTML editor on the UI where the administrator can define a **single, global HTML email design**. All emails (welcome and reactivation) will use this template.

The template relies on specific shortcodes. The plugin will dynamically replace these with content based on the active email type. If a shortcode is not relevant for a specific email, the plugin will automatically remove it from the final HTML before sending.

**Available Email Shortcodes:**
* `[WMA_MESSAGE]` - Inserts the custom text message configured for the specific email type. Always rendered.
* `[WMA_REVIEW_PRODUCTS]` - Inserts the HTML table of products from the triggering order. Rendered only when `include_review_products = true` on the email record. Always stripped for the welcome email (no order context).
* `[WMA_DISCOUNT_PRODUCTS]` - Inserts the HTML table of the top discounted products. Rendered only when `top_sale_products > 0` on the email record. Always stripped for the welcome email.
* `[WMA_COUPON_CODE_PERCENT]` - Inserts a generated percentage discount WooCommerce coupon code. Rendered only when `coupon_percent > 0` on the email record (or shortcode param for welcome email).
* `[WMA_COUPON_CODE_FREESHIPMENT]` - Inserts a generated free shipping WooCommerce coupon code. Rendered only when `coupon_expiry_freeship > 0` on the email record. Not available for the welcome email.

**Stripping rule:** When a shortcode evaluates to empty (its condition is not met), the plugin removes the shortcode tag and its generated HTML from the final email before sending.

**Unsubscribe Link:** The plugin will automatically append the required List-Unsubscribe headers and the footer unsubscribe link to all outgoing emails.

### Feature: Sendy subscription (Frontend)

Input fields on the UI (Sendy Settings tab):

* `SENDY_URL`     | Sendy installation URL - mandatory
* `SENDY_API_KEY` | Sendy API key - mandatory

Sendy forms are deployed via shortcodes:

```
[wma-sendy id="unique-form-id" list="listId" redirect="redirect-url" coupon_percent="10" coupon_expiry="7"]
```
* `id` (mandatory): form ID (must be used as a prefix/namespace in the generated HTML to prevent DOM ID collisions).
* `list` (mandatory): Sendy list ID.
* `redirect` (optional): visitor will be redirected to this URL after successful subscription.
* `coupon_percent` (optional): a coupon will be created with this percent amount and sent to the new subscriber when set.
* `coupon_expiry` (optional): coupon expiry period in days (when `coupon_percent` is set, default value: 30).

**Welcome Email / Subscription Confirmation:**
A dedicated tab on the UI should allow configuring the email sent upon successful subscription:
* `welcome_email_subject` | Subject of the welcome email
* `welcome_email_message` | The content injected into the `[WMA_MESSAGE]` shortcode. 

### Feature: Adding new customers to different lists

When a new order is completed (Status: Completed), the name and e-mail address of the new customer is added to a Sendy list asynchronously.

The UI should provide 3 fields for these settings:
* `customers_list`           | every customer will be added to this list | mandatory field
* `vip_customers_list`       | customers above an order amount will be added to this list | optional field
* `vip_customers_amount`     | order value, ie: 20000 | optional field (unavailable when `vip_customers_list` is empty)
* `returning_customers_list` | if a customer is already added to `customer_list` they will be added to `returning_customers_list` also | optional field

**Returning Customer Logic:** To determine if a customer is returning, the plugin must query the WooCommerce database to check if there is any *previous* successful order associated with the same billing email address. 

Logic: Every customer is added to `customers_list`. VIP customers are added to `vip_customers_list` and `customers_list` too. Returning customers are added to `returning_customers_list` and `customers_list`.

### Feature: Dynamic Reactivation Email Management

Replaces the legacy `review-email.php`, `refill-email.php`, `forsale-email.php`, `final-email.php` scripts with a flexible, user-configurable system.

Administrators can create, edit, enable/disable, and delete any number of reactivation emails. The admin UI follows the WooCommerce shipping methods pattern: a list page with add/edit/delete/toggle actions.

**On plugin activation**, 4 default reactivation emails are pre-populated:

| # | Name | `wait_period` | `include_review_products` | `top_sale_products` | `coupon_percent` | `coupon_expiry_percent` | `coupon_expiry_freeship` |
|---|------|--------------|--------------------------|---------------------|-----------------|------------------------|--------------------------|
| 1 | Review Request | 7 | `true` | `0` | `0` | `30` | `0` |
| 2 | Refill Offer | 30 | `false` | `0` | `0` | `30` | `7` |
| 3 | For-Sale Products | 60 | `false` | `10` | `0` | `30` | `0` |
| 4 | Final Notification | 90 | `false` | `10` | `10` | `7` | `0` |

**Each reactivation email record has the following fields:**

| Field | Description | Shortcode controlled |
|-------|-------------|----------------------|
| `name` | Admin-visible label (e.g., "Review Request") | — |
| `enabled` | Enable/disable toggle | — |
| `wait_period` | Days after order completion before sending | — |
| `email_subject` | Subject line of the email | — |
| `email_message` | Content injected into `[WMA_MESSAGE]` | `[WMA_MESSAGE]` |
| `include_review_products` | `true` = render ordered products table, `false` = strip | `[WMA_REVIEW_PRODUCTS]` |
| `top_sale_products` | Number of top discounted products to show; `0` = strip | `[WMA_DISCOUNT_PRODUCTS]` |
| `coupon_percent` | Percentage discount value; `0` = strip | `[WMA_COUPON_CODE_PERCENT]` |
| `coupon_expiry_percent` | Expiry in days for the percentage coupon (default: 30) | `[WMA_COUPON_CODE_PERCENT]` |
| `coupon_expiry_freeship` | Expiry in days for the free shipping coupon; `0` = strip | `[WMA_COUPON_CODE_FREESHIPMENT]` |

**State tracking:** Each email tracks its sent state in WooCommerce order meta using the key `_wma_email_{id}_sent` where `{id}` is the email's unique integer ID. This prevents duplicate sends per order.

**Cron:** A single daily WP-Cron event iterates over all enabled reactivation emails and dispatches any due sends.

**Unused shortcodes** in the global template are automatically stripped before sending, so the same global template works for all email types regardless of which shortcodes each email actually uses.

### Multi language support

Create 2 `.po` files: english and hungarian language

Every label and title on the plugin UI and texts in the form generated by the shortcode should use the texts from the `.po` file.

`.po` file selection: Wordpress default language. If it is not english or hungarian then use the english one.
